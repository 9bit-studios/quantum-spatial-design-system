## Figma MCP Workflow

### Step 1: Build Your Designs

1. **Use your component library** - Don't create one-off elements
2. **Apply variables consistently** - Colors, spacing, typography
3. **Name layers clearly** - "Button Primary", not "Rectangle 47"
4. **Create variants properly** - For different states/sizes
5. **Use Auto Layout everywhere** - Critical for responsive behavior

### Step 2: Prepare for MCP

**Best Practices:**

- âœ… **Flatten complex vectors** into single shapes where possible
- âœ… **Use component instances**, not detached copies
- âœ… **Apply constraints** for responsive behavior
- âœ… **Group related elements** with clear names
- âœ… **Add descriptions** to key components

### Step 3: Using Figma MCP in Claude

**Workflow:**

1. **Open Figma file** in desktop app
2. **Select the frame/component** you want to export
3. **In Claude Desktop**, invoke Figma MCP:
    - `get_design_context` - Gets code for selected node
    - `get_metadata` - Gets structure overview
    - `get_screenshot` - Gets visual preview

**Example Claude Prompt:**

```
"Generate React code for the currently selected
Executive Dashboard frame in Figma. Use Tailwind
for styling and ensure it's responsive."

```

**What MCP Returns:**

- Component structure (HTML/JSX)
- Styles (CSS/Tailwind classes)
- Layout properties
- Variable references

### Step 4: Code Generation Tips

**For best results:**

1. **Select parent frames**, not nested elements
2. **Test one component at a time** initially
3. **Specify your framework** (React, Vue, HTML)
4. **Request specific libraries** (Tailwind, styled-components)
5. **Iterate** - Refine designs based on code output

### Design Efficiency

1. **Copy styles from HTML** - Use browser inspector to validate exact values
2. **Create component sets** - Not individual components for each variant
3. **Use plugins**:
    - "Content Reel" - For realistic member data
    - "Chart" - For data visualizations
    - "Iconify" - For consistent icons
    - "Stark" - For accessibility checks

### Figma MCP Optimization

1. **Keep frames flat** - Deeply nested groups slow MCP processing
2. **Use meaningful names** - "Header/Title" not "Text Layer 23"
3. **Leverage variants** - MCP recognizes component variants
4. **Test incrementally** - Don't wait until everything's done

### Presentation Strategy

1. **Create user flows** - Show journeys, not just static screens
2. **Add interaction annotations** - "What happens when..."
3. **Include metrics prominently** - Steve cares about ROI
4. **Show before/after** - Demonstrate AI impact visually
